# ServiceNow Ticket Automation Project

## Overview
Built a custom ServiceNow application to automate IT service requests. The goal was to reduce manual work by automatically creating tasks and updating ticket statuses based on workflow logic.

---

## What It Does
- Users submit requests through a service catalog  
- Tasks are automatically created from those requests  
- Ticket status updates to **"In Progress"** once work begins  
- Simulates a real IT support workflow from start to finish  

---

## Why I Built This
In many IT environments, handling tickets manually slows things down.  
This project simulates how companies use ServiceNow to:
- Reduce manual effort  
- Standardize workflows  
- Improve efficiency in IT operations  

---

## How It Works
1. A user submits a request through a form  
2. A record is created in ServiceNow  
3. A business rule runs automatically  
4. A task is generated and assigned  
5. The request status updates based on the workflow  

---

## Tools & Features Used
- ServiceNow (Creator Studio)  
- Service Catalog & Record Producers  
- Business Rules (automation logic)  
- UI Policies & Client Scripts  
- ITSM concepts (request lifecycle, task management)  

---

## What I Focused On
- Automating repetitive tasks  
- Designing simple but realistic workflows  
- Understanding how IT service processes are structured  
- Reducing manual intervention using platform logic
